\# Shoe Sell-Through Predictive Model



\## Project Overview



This project builds a predictive model for sneaker sell-through rates at the store level.

The goal is to move from reactive inventory management to data-driven decision making —

identifying which styles are likely to underperform before they become a problem.



\## Core Outputs



\- Predicted sell-through percentage per shoe style

\- Risk flags for styles unlikely to sell through before markdown or return window

\- Reorder and discount recommendations based on predicted performance



\## Data



This project uses mock/sample data modeled after real retail inventory and sales patterns.

Data lives in the `/data` folder. Schema is defined in `/sql/schema.sql`.



\## Project Phases



1\. \*\*SQL Schema\*\* — Define tables for inventory, sales transactions, and product attributes

2\. \*\*Python Model\*\* — Build predictive model using sell-through rate as the target variable

3\. \*\*Jupyter Notebook\*\* — Document analysis, visualizations, and model results

4\. \*\*Business Case\*\* — Frame findings for a retail or corporate audience
